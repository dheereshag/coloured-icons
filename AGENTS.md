# Coloured Icons — Agent Instructions

A Next.js showcase app for the **Coloured Icons** CSS icon library, which is served directly from GitHub via jsDelivr CDN. The library lets users embed brand/tech/animal icons with a simple `<i class="ci ci-react ci-2x">` HTML tag.

## Commands

```bash
pnpm dev        # generate logos.css then start Next.js dev server
pnpm build      # generate logos.css then build for production
pnpm lint       # ESLint
pnpm minify     # minify app/logos.css → app/logos.min.css and app/ci.css → app/ci.min.css
```

> **Important:** `pnpm dev` and `pnpm build` both run `node scripts/listFolders.js` first, which regenerates `app/logos.css` from the file system. You rarely need to run it manually.

## Architecture

```
public/logos/<category>/<icon-name>/    ← icon SVG/PNG assets
scripts/listFolders.js                  ← generates app/logos.css by scanning public/logos/
app/logos.css                           ← AUTO-GENERATED, never edit manually
app/logos.min.css                       ← AUTO-GENERATED, never edit manually
app/ci.css                              ← hand-crafted base styles (.ci, sizing classes)
app/ci.min.css                          ← AUTO-GENERATED by pnpm minify
constants/icons.ts                      ← manually maintained Icon[] list (name, category, classes, url)
constants/logoAliases.js                ← alternate class names (e.g. "fb" for facebook)
constants/logoMeta.js                   ← display name overrides for icons with special chars
```

### Data flow

1. SVG files live in `public/logos/`
2. `listFolders.js` scans the tree and writes CSS rules like `.ci-react { content: url(...) }` to `app/logos.css`
3. The Next.js app imports that CSS and exposes a searchable icon gallery powered by `constants/icons.ts`

## Adding a New Icon

1. Create folder: `public/logos/<category>/<icon-name>/`
2. Add asset files following the naming convention below
3. Add an entry to `constants/icons.ts` (`name`, `category`, `classes[]`, `url`)
4. Optionally add aliases in `constants/logoAliases.js`
5. Run `pnpm dev` or `pnpm build` — CSS regenerates automatically

### File naming convention

| File | Purpose |
|------|---------|
| `<name>.svg` | Default / dark-mode variant |
| `<name>-light.svg` | Light-mode variant |
| `<name>-horizontal.svg` | Wordmark + icon side-by-side |
| `<name>-horizontal-light.svg` | Light version of horizontal |
| `<name>-vertical.svg` | Wordmark above/below icon (also aliased as `-stacked`) |
| `<name>-wordmark.svg` | Text-only wordmark |

### CSS class pattern (generated automatically)

```html
<i class="ci ci-react ci-2x"></i>           <!-- default -->
<i class="ci ci-react-light ci-lg"></i>     <!-- light variant -->
<i class="ci ci-react-horizontal"></i>      <!-- horizontal wordmark -->
<i class="ci ci-react ci-invert"></i>       <!-- invert colours -->
```

Sizing classes mirror Font Awesome: `ci-xs`, `ci-sm`, `ci-md`, `ci-lg`, `ci-xl`, `ci-2xl`, `ci-1x`–`ci-10x`.

## Conventions

- **Path alias**: `@/` resolves to the project root (configured in [tsconfig.json](tsconfig.json))
- **UI components**: shadcn/ui in `components/ui/`. Use `cn()` from `lib/utils.ts` for conditional classes
- **Component folders**: each feature has its own subfolder (`filter/`, `modal/`, `navbar/`, `search/`, `home/`, `about/`) with a barrel `index.ts`
- **JS constants** (`logoAliases.js`, `logoMeta.js`): plain `.js` ESM because they are imported by the Node script as well as the Next.js app — do not convert them to TypeScript
- **No test suite** is configured; verify changes with `pnpm build` and `pnpm lint`
- **Tailwind v4** (PostCSS plugin, not v3 config file)
